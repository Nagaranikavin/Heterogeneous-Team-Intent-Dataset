# Dataset Generation## 1. OverviewThis dataset was developed to support research on team-level intent prediction in coordinated multi-agent environments. It captures heterogeneous air combat missions in which multiple aircraft cooperate to accomplish a common objective while executing different tactical roles. The dataset provides raw aircraft trajectories together with extracted node and edge features for graph-based machine learning models.---## 2. Simulation EnvironmentThe dataset was generated using the **SCRIMMAGE (Scalable Robotic Interactive Multi-Agent Game Engine)** simulator. SCRIMMAGE is an open-source, physics-based multi-agent simulation framework that enables realistic modeling of autonomous aerial vehicles, mission planning, and agent interactions. It provides synchronized state information for all agents throughout the simulation, making it suitable for generating temporal multi-agent datasets.---## 3. Mission DesignFour representative air combat missions were designed to capture different forms of coordinated team behavior:- **Sky Sentinel Strike (SSS)** – A coordinated strike mission involving an attacker, two flankers, and a decoy.- **Escort Penetration Strike (EPS)** – A mission in which multiple escort aircraft protect a penetrator during its approach toward the target.- **Anti-Ship Strike (ASS)** – A low-altitude strike mission involving coordinated attack, flanking, and deception against a naval target.- **Reconnaissance (Recon)** – A surveillance mission in which reconnaissance aircraft collaboratively explore, detect, and track targets.Each mission represents a distinct coordination strategy, resulting in diverse interaction patterns among the participating agents.---## 4. Scenario GenerationThe complete dataset consists of **74 simulation scenarios**. Each scenario was generated by varying the initial conditions of the participating aircraft, including their positions, headings, velocities, and formations. These variations create diverse mission executions while preserving the tactical objectives of each mission type, allowing machine learning models to learn robust coordination patterns rather than memorizing fixed trajectories.---## 5. Mission ExecutionEach simulation progresses through mission-specific operational phases. During execution, aircraft continuously adapt their maneuvers according to their assigned tactical roles while coordinating with teammates to accomplish the mission objective. The resulting trajectories capture both individual agent behavior and collective team coordination throughout the mission.---## 6. Trajectory RecordingDuring simulation, the state of every aircraft is recorded at each simulation time step. The recorded information includes the aircraft's spatial position, motion characteristics, and other flight parameters required to reconstruct the complete mission trajectory. These recorded trajectories form the raw dataset from which graph-based features are extracted.---## 7. Feature ExtractionThe recorded trajectories are processed to generate graph-based representations suitable for machine learning. Individual aircraft states are converted into **node features**, while pairwise relationships between aircraft are represented as **edge features**. These features capture both the individual dynamics of each aircraft and the interactions among team members. A detailed description of all node and edge features is provided in the `metadata` folder.---## 8. Dataset OrganizationFor each simulation scenario, the dataset is organized into three components:- **Raw trajectories** containing the original aircraft states generated by the simulator.- **Node features** representing the state and motion characteristics of individual aircraft.- **Edge features** representing the pairwise interactions between aircraft.This organization enables researchers to directly use either the raw simulation data or the processed graph representations depending on their application.---## 9. Representative SamplesTo keep this GitHub repository lightweight and easy to access, representative sample scenarios are included. These samples preserve the original data structure, feature organization, and file formats used throughout the complete dataset. The full dataset consists of **74 simulation scenarios**, each containing **500 temporal observations**, and was used for all experiments reported in the associated research work.
